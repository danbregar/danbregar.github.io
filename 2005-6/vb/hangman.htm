Hangman

Visual Basic

 

Create a hangman game.  There should be a textbox that allows one player to type in a word.  There should be a “Select Word” button that, when pressed, stores the current word in a module-level variable, clears the textbox, and disables the textbox and the “Select Word” button.

 

There should be a scaffold that starts off empty below this textbox.

 

To the right of the scaffold, there should be another textbox that allows the other player to type in a letter.  If the letter is in the word, it should show up below the scaffold in the right spot for the word.  If the letter is not in the word, it should show up above the scaffold, and a stick figure should be drawn on the scaffold (make the body appear part by part).  Once all the parts of the stick figure appear, the second player has lost – there should be a message displayed in a label and the game should start over.

 

There should be buttons allowing either player to restart or quit the game at any time. (The “Esc” key should also automatically quit the game).

 

This program requires you to use For/Next loops (p. 256 – 261) and know how to use different string functions (p. 643).

 

You will be graded based on the following criteria:

a.   Properly formatted variable names and object names. (5 points)

b.   Properly commented code and variables. (5 points)

c.   Proper user interface; Escape key works as expected. (5 points)

d.   Proper code functioning – the program works.  (15 points)
